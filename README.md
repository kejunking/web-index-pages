# web-index-pages

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为通用 Web 页面文件的存储与展示空间。

## 项目简介

本仓库主要用于存放各类独立的 HTML 页面文件，这些页面可以包含信息展示、工具界面、实验性页面或其他静态内容。所有页面均为独立的 HTML 文件，不依赖特定后端服务或数据库。

## 目录说明

```
web-index-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面所需的静态资源（CSS、JS、图片等）
├── index.html      # 仓库首页，可列出页面索引
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件代表一个独立页面，文件名应具有描述性。
- `assets/`：按需组织，可将公共资源放在子目录中。
- `index.html`：可选，用于提供页面导航或展示列表。

## 页面归档说明

所有归档的 HTML 页面应遵循以下规范：

- 每个页面应独立完整，包含必要的 HTML 结构。
- 页面内可引用 `assets/` 目录下的资源文件。
- 文件名建议使用英文小写字母、数字及连字符（`-`），避免空格和特殊字符。
- 页面标题（`<title>`）应与文件名或内容主题相关。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并更新 `index.html`（如有）中的索引链接。
- 修改现有页面时，请保持文件路径不变，避免破坏已有链接。
- 删除页面时，建议同时清理 `index.html` 中的相关链接及不再使用的资源文件。
- 如有重大结构调整，请同步更新本 README 文件。
